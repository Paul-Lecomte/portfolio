Why I Built It

I wanted to create a practical application that solves a real-world business problem: managing inventory efficiently. Stocker was built to learn and demonstrate full-stack development capabilities ‚Äî from building a secure REST API to crafting a responsive frontend with React.

REST API Overview

Stocker provides a secure RESTful API to manage users, products, categories, and inventory logs. Example endpoints include:

  • POST /api/auth/login ‚Äì User login
  • GET /api/products ‚Äì Fetch all inventory items
  • POST /api/products ‚Äì Add new product
  • PUT /api/products/:id ‚Äì Update product
  • DELETE /api/products/:id ‚Äì Remove product

All endpoints are protected using JWT-based authentication and role verification.

Authentication & Authorization

Authentication is handled using JSON Web Tokens (JWT), enabling secure session management. Each token includes a user ID and role.

Authorization logic restricts access to admin-only routes, like creating or deleting inventory items.
