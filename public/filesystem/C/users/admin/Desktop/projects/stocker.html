Database Schema

The database schema of Stocker is designed for scalability and flexibility, supporting features like real-time updates, role-based access control, and detailed product management. Below is a breakdown of the key tables:

Users

    Table: users
    Columns:
        - id (primary key)
        - username (unique)
        - email (unique)
        - password_hash
        - role (admin/user)
        - created_at
        - updated_at

Products

    Table: products
    Columns:
        - id (primary key)
        - name
        - description
        - quantity
        - price
        - category_id (foreign key to categories)
        - created_at
        - updated_at

Categories

    Table: categories
    Columns:
        - id (primary key)
        - name
        - description

Stock Movements

    Table: stock_movements
    Columns:
        - id (primary key)
        - product_id (foreign key to products)
        - movement_type (addition/subtraction)
        - quantity
        - created_at
        - updated_at

Notifications

    Table: notifications
    Columns:
        - id (primary key)
        - user_id (foreign key to users)
        - message
        - read_status (boolean)
        - created_at
